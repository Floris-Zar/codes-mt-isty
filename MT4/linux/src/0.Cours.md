# Notes de cours importantes

## Définitions

- **Processus** : instance d'un programme en cours d'exécution, chaque processus a un PID unique
- **Coeur** :       unité de traitement d'un processeur, un processeur peut avoir plusieurs coeurs
- **CPU** :         unité de traitement d'un ordinateur, un ordinateur peut avoir plusieurs processeurs
- **Signal** :      mécanisme de communication entre processus, un signal est un message envoyé à un processus pour lui indiquer qu'un événement s'est produit
- **Symétrie** :     relation entre deux processus qui font les mêmes choses, mais dans un ordre différent
- **Processus zombie** : processus qui a terminé son exécution, mais dont le père n'a pas encore récupéré le statut de terminaison, ce qui laisse une entrée dans la table des processus
- **Processus orphelin** : processus dont le père a terminé son exécution
- **Processus init** : processus avec PID 1, qui est le père de tous les processus orphelins, il est responsable de récupérer le statut de terminaison des processus zombies

## Élements de code

`sysconf(_SC_NPROCESSORS_ONLN)` : renvoie le nombre de processeurs disponibles sur le système
`#define _GNU_SOURCE` : pour utiliser `sched_getcpu()` qui renvoie le numéro du CPU sur lequel le processus s'exécute actuellement
 
`getpid()` : renvoie le PID du processus courant
`getppid()` : renvoie le PID du processus parent

`fork()` : crée un nouveau processus en dupliquant le processus courant.
- processus parent : fork() renvoie le PID du processus fils
- processus créé (fils) : fork() renvoie 0
- en cas d'erreur : fork() renvoie -1

`system("date +\"%H:%M:%S:%N\"");` : affiche l'heure actuelle au format heure:minute:seconde:nanoseconde

`alarm(10)` : programme une alarme pour qu'elle se déclenche après 10 secondes, en envoyant le signal SIGALRM au processus courant (càd le processus qui a appelé alarm())


## Différents signaux

La différence entre les signaux utilisateurs et les signaux système est l'émetteur du signal:
- les signaux utilisateurs sont envoyés par des processus (ex: kill -10 <pid> ou raise(SIGUSR1))
- les signaux système sont envoyés par le système d'exploitation
**Attention** : nécessite de rajouter `#include <signal.h>` pour les utiliser

### Signaux utilisateurs

- `SIGUSR1` ou `SIGUSR2` : signaux utilisateurs 1 et 2, signaux envoyés au processus par un autre processus (ex: kill -10 <pid>) ou par le processus lui-même (ex: raise(SIGUSR1)), par défaut ces signaux terminent le processus, mais il est possible de les gérer pour faire autre chose (ex: afficher un message, ou ignorer le signal)

### Signaux système

- `SIGALRM` : signal envoyé au processus lorsque l'alarme programmée par alarm() se déclenche, par défaut ce signal termine le processus, mais il est possible de le gérer pour faire autre chose (ex: afficher un message, ou ignorer le signal)
- `SIGSTOP` : signal envoyé au processus pour le suspendre, par défaut ce signal suspend le processus, mais il est possible de le gérer pour faire autre chose (ex: afficher un message, ou ignorer le signal)
- `SIGCONT` : signal envoyé au processus pour le reprendre après une suspension, par défaut ce signal reprend le processus, mais il est possible de le gérer pour faire autre chose (ex: afficher un message, ou ignorer le signal)
